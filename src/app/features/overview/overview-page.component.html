Getting started

Angular Interview Structure

This project demonstrates key Angular concepts through a hands-on example. Follow the interview flow to learn how modern Angular handles state, routing, and component architecture.

Start Interview Flow Angular Docs

What You'll Learn

  • Bootstrap: How main.ts and app.config.ts set up the app.
  • Signals & Computed: Modern state management without RxJS boilerplate.
  • Lazy Routes: Load features on demand for better performance.
  • Components & DI: Component hierarchy and dependency injection patterns.
  • Templates: Native control flow with @for, @if instead of legacy directives.

Project Structure

File Organization

  • src/app/core/

    Shared models, data, services.

  • src/app/features/

    Feature modules (interview, overview) lazy-loaded by route.

  • src/app/app.ts

    Root shell component with navigation.

  • src/app/app.routes.ts

    Route configuration with lazy loading.

Key Concepts

What to Mention in Interview

Signals

Local reactive state that updates components efficiently with OnPush.

Services

Singleton logic centralized in services, injected via dependency injection.

Lazy Loading

Features loaded only when their route is accessed, reducing initial bundle.

Components

Small, focused units with clear input/output boundaries.
